# Open Manufacturing Engineering Platform (OpenMEP)

OpenMEP is a lightweight open-source web platform for manufacturing process engineering, factory layout design, process modeling, resource management, and discrete-event simulation.

The project is intended for small and medium-sized manufacturing companies, students, researchers, industrial engineers, and independent manufacturing engineering practitioners.

## Current MVP Status

Implemented modules:

- Project Manager CRUD
- Layout Designer v1 with Konva.js canvas and MySQL persistence
- Resource Manager v1 with project-scoped resources and optional layout linking
- Process Designer v1 with validation and MySQL persistence
- Simulation Engine v1 with stored KPI results
- Results Dashboard v1

## Architecture

OpenMEP follows a layered PHP architecture:

```text
Browser → JavaScript → REST API → Controller → Service → Repository → MySQL → JSON Response → UI Update
```

Core rules:

- Controllers validate requests and call services.
- Business logic belongs in services.
- Repositories perform database operations.
- JavaScript modules manage UI state and API calls.
- All API endpoints return JSON.

## Installation

1. Create a MySQL database named `openmep` by importing the schema:

```bash
mysql -u root -p < database/schema.sql
```

2. Optional: import demo data:

```bash
mysql -u root -p openmep < database/seed_demo.sql
```

3. Copy the configuration file:

```bash
cp config/config.example.php config/config.php
```

4. Update database credentials in `config/config.php`.

5. Start a local PHP server:

```bash
php -S localhost:8000
```

6. Open the application:

```text
http://localhost:8000
```

## Shared UI Framework

OpenMEP includes a lightweight shared UI layer for routing, status display, toast notifications, confirmation dialogs, validation helpers and API lifecycle events. See `docs/SHARED_UI.md`.


## Validation Engine

OpenMEP validates process models before simulation. Operation, Inspection and Transport nodes require assigned resources, valid cycle times and graph connectivity. See `docs/VALIDATION.md`.


## Simulation Engine v2

The simulation backend now uses a deterministic event-queue execution model with FIFO queues, finite resource capacity, routing probabilities, scrap/rework handling and extended KPI metadata. See `docs/SIMULATION_ENGINE.md`.


## Data Exchange

OpenMEP includes project package import/export for sharing engineering models between installations. Exported packages contain project settings, layout elements, resources, operations, process connections, resource assignments and simulation scenarios. Simulation results can be exported separately as CSV.

See [`docs/IMPORT_EXPORT.md`](docs/IMPORT_EXPORT.md) for details.
